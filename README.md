# AI Web Code Generator

基于 AI 的 Web 代码生成平台，使用智谱 GLM 大模型生成完整的 Web 项目代码（前端 + 后端），支持一键 Docker 部署。

## 功能特性

- **AI 代码生成**：通过自然语言描述生成完整的前后端代码
- **代码审查与修复**：AI 自动审查并修复生成的代码错误
- **代码编辑**：在线编辑修改生成的代码
- **代码修改**：根据用户反馈修改代码（修复 bug、添加功能等）
- **会话管理**：保存历史对话和代码，支持继续会话
- **一键部署**：支持前后端分离的 Docker 部署
- **部署管理**：查看、启动、暂停、删除部署

## 技术栈

- **前端**：Vue 3 + Element Plus + TypeScript + Vite
- **后端**：FastAPI + SQLAlchemy
- **数据库**：MySQL
- **AI**：智谱 GLM 大模型

## 快速开始

### 本地开发

1. **启动后端**

```bash
cd backend
pip install -r requirements.txt
python main.py
```

2. **启动前端**

```bash
cd front
npm install
npm run dev
```

3. **访问**

- 前端：http://localhost:5173
- 后端 API：http://localhost:8000

### Docker 部署

```bash
docker-compose up -d
```

访问 http://localhost

## API 文档

启动后端后访问：http://localhost:8000/docs

## 使用流程

1. 注册/登录账号
2. 在输入框描述想要的页面（如"创建一个登录页面"）
3. AI 生成完整的前后端代码
4. 可以编辑代码、查看预览
5. 根据反馈修改代码（如"把按钮改成红色"）
6. 一键 Docker 部署

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              代码生成流程                                        │
└─────────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────┐
    │  用户输入需求   │
    │ "创建一个登录页面" │
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  调用 GLM 大模型 │
    │  generate_code  │
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  AI 生成代码    │
    │ (JSON 数组格式) │
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐     ┌─────────────────┐
    │ JSON 解析成功?  │────▶│      否         │
    └────────┬────────┘     └─────────────────┘
             │ 是
             ▼
    ┌─────────────────┐
    │  添加 folder 字段 │
    │ (frontend/backend) │
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  调用代码审查   │
    │  review_code    │
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐
    │  JSON 验证成功? │
    └────────┬────────┘
             │
       ┌─────┴─────┐
       │           │
       │ 是        │ 否
       ▼           ▼
   ┌───────┐  ┌───────────┐
   │ 返回  │  │ 返回原代码 │
   │ 审查后 │  └───────────┘
   │ 代码  │
   └───────┘
```
在新会话的第一条记录，会调用生成代码接口生成代码。

流程：生成代码的功能流程，用户输入需求 --> 大模型生成代码 --> JSON格式校验与改正 --> 大模型review代码。

改进：可以重写用户需求，让用户的需求更加准确，引导大模型更好回答。在大模型生成代码和review代码，可以通过多模型合作、RAG和训练专门模型等办法提高效果。在校验这部分，不仅仅是校验json格式、还可以调用专门的工具校验语法、词法等工具校验，让大模型更好修复。

在新会话的第一条记录之后，可根据用户反馈修改代码（例如：bug、新需求和优化等），然后会调用修改代码接口生成代码。

改进：可参考生成代码的改进方法。


### 部署
点击Docker部署可直接部署代码，目前已经可以成功部署前后端代码并运行起来。
但是前端暂时没法访问后端，需要解决。
部署的脚本也是把前后端的运行端口写死，后面需要调用大模型等方法，识别前后端的运行端口，改进部署脚本。
如果部署失败，会返回报错信息，用户可根据报错信息要求修改代码。后期部署失败，可以把报错信息和代码喂给大模型，让大模型修改代码或部署脚本，从而成功部署应用。
